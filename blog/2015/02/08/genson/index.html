genson

Truly own your blog

genson is a static blog generator written in Python. It couldn't be easier to use!

Simply write your blog posts in easy-to-use Markdown, then do python genson.py and a

blog directory tree magically appears!

Just upload to your favourite web host, and you're done.

Why genson? My surname's Johnson, and this is a _gen_erator. Funny, huh?

Features

  1. Uses easy Markdown files to create perfect HTML.
  1. Organizes your blog in a neat directory tree.
  1. Auto-creates a table of contents and blog front page.
  1. Incorporates existing templates and maintains links to JavaScript and CSS.

Requirements

  1. Python 2.*
  1. Ability to cd in the Terminal.

Instructions

  1. Keep the file genson.py in the root directory of your website.
  1. Write a blog post in Markdown. Try it out here.
  1. The first line of your post must be the title.
  1. You can include the dates and times of creation and editing of the post by entering

08-02-2015 at 11:50 PM and 08-02-2015 at 11:50 PM where you want them on the page.

  1. Indicate what part, if any, you want to be the preview on your front page with two

lines, one saying , the other.

  1. Save your post with the Markdown file extension, .md, in the root directory of your

website.

  1. Gather any HTML templates (if using them) and their resources (JS and CSS) into the

root directory, and name them template.html, template.js, and template.css.

  1. In the HTML template, replace the JS and CSS paths with JS_FILE and CSS_FILE.
  1. In the HTML template, where you want the blog post to go, enter INSERT_POST_HERE.
  1. In the terminal, cd to the root directory of your website.
  1. If you're using the templates, do python genson.py -t template.
  1. If not using a template, do python genson.py.
  1. If the last line in the Terminal was Success!, then everything should have gone

well.

  1. Voila! You should have a new directory called blog containing the Table of Contents

in toc.html, the front page in index.html, and your individual posts in the

sub-directories.

Advanced use

Try python genson.py -h to see more advanced options.

The locations of the Markdown files and templates can be specified, as well as the name of

the blog root directory.
